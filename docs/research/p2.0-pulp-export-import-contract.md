## Scope

Prove the native Pulp core export/import contract for the local baseline image
and CLI tooling, with apt/deb repository viability as the target.

## Native Command Surface (pulp-cli)

### Exporters

- Group: `pulp exporter pulp`
- Verbs observed: `create`, `destroy`, `list`, `show`, `update`
- Create contract:
  - required: `--name`, `--path`
  - repository binding: `--repository` (typed ref) or `--repository-href`
  - relationship: exporter owns repository references used by exports

### Exports

- Group: `pulp export pulp`
- Verbs observed: `run`, `list`, `show`, `destroy`
- Run contract:
  - required: `--exporter`
  - optional: `--full`, `--chunk-size`, `--versions`, `--start-versions`
  - relationship: export run is launched under an existing exporter

### Importers

- Group: `pulp importer pulp`
- Verbs observed: `create`, `destroy`, `list`, `show`, `update`
- Create contract:
  - required: `--name`
  - optional map: `--repo-map <source_repo> <dest_repo>`
  - relationship: importer stores repository mapping context for imports

### Import-check and Imports (CLI status)

- `pulp import-check --help` → **No such command** (exit 2)
- `pulp import --help` → **No such command** (exit 2)
- `pulp importer pulp import-check --help` → **No such command** (exit 2)
- `pulp importer pulp import --help` / `imports --help` → **No such command** (exit 2)

## API vs CLI Gap (Hard Blocker)

The selected image exposes import endpoints in OpenAPI:

- `/pulp/api/v3/importers/core/pulp/import-check/` (POST)
- `{pulp_importer_href}imports/` (GET/POST)

However, the pinned `pulp-cli 0.37.1` surface in this repo does not expose
native CLI verbs for import-check/import execution. For this baseline, strict
"native `pulp-cli` only" high-side import automation is blocked.

## Apt/Deb Export/Import Viability Conclusion

1. **Apt repositories are export-capable at the plugin layer.**  
   `pulp_deb 2.20.4` satisfies the pulpexport capability gate (`>=2.20.0`).
2. **Generic exporter/export command groups are present and usable in CLI.**
3. **Current baseline is blocked for end-to-end rehearsal by defaults + CLI gap:**
   - `ALLOWED_EXPORT_PATHS=[]` and `ALLOWED_IMPORT_PATHS=[]` in container settings.
   - exporter create fails fast with:
     `Path '/var/lib/pulp/exports/...' is not an allowed export path`.
   - import-check/import operations are present in API but not available as
     native `pulp-cli` verbs in this pinned CLI.

## Version-Gated / Unsupported Notes

- `--href` help on exporter/importer includes: "accepts PRNs if core>=3.63";
  baseline core is 3.21.x, so PRN behavior is out-of-scope.
- Deb sync `--optimize/--no-optimize` requires `pulp_deb >= 2.20.0`; baseline
  satisfies this gate.
- Import-check/import command verbs are unsupported in pinned CLI baseline
  (`pulp-cli 0.37.1`) despite API presence.

## Fail-Fast Failure Modes Observed

1. `pulp exporter pulp create --repository local-apt-smoke-repo ...` fails with:
   `A resource type must be specified with the repository option.`
2. `pulp exporter pulp create --path /var/lib/pulp/exports/...` fails with:
   `Path '...' is not an allowed export path` when allowed paths are empty.
3. `pulp import-check` / `pulp import` fail immediately as unknown commands.

## Next Implementation Hooks (Low/High Harness)

1. **Low-side preflight gate (P2.2 prerequisite):**
   - capture and assert `ALLOWED_EXPORT_PATHS` non-empty before exporter create.
   - fail closed with explicit remediation if empty.
2. **Low-side export contract wiring:**
   - keep immutable `latest_version_href` capture (already available).
   - bind exporter to repository href (`--repository-href`) to avoid typed-name
     ambiguity.
3. **High-side import contract gate:**
   - require a validated image+CLI combination where native `pulp-cli` exposes
     import-check/import verbs before implementing P2.2/P2.3 import automation.
4. **P2.1 manifest alignment:**
   - include `path`, `toc`, and importer repo-map assumptions as explicit fields
     so they map directly to native Pulp import APIs when CLI support is present.
